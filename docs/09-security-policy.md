# セキュリティ方針

このプロジェクトでは、学習のために脆弱な状態を作る場合があります。安全な範囲を明確にして検証します。

## 基本的な注意事項

このリポジトリでは、意図的に脆弱なコードを書くことがあります。

必ず次を守ります。

- localhostでのみ動作させる
- 脆弱な状態でインターネットへ公開しない
- 本物のパスワードを使わない
- 実在する個人情報を使用しない
- 本物のAPIキーを使用しない
- `.env`をGitへコミットしない
- セッションIDをログへ出さない
- パスワードをログへ出さない
- 他人が管理するWebサイトへBurp Suiteを使用しない
- 許可されていない対象へ攻撃を行わない
- 学習用の脆弱な実装を本番コードへ流用しない

---

## 検証環境

- 原則としてlocalhostで実行する
- 外部公開が必要な検証は、この学習プロジェクト本体では行わない
- 脆弱な状態と修正済み状態をブランチやコミットで区別する
- テストデータだけを使用する
- アップロードファイルを不用意に実行しない

## 秘密情報

Gitへ含めないもの：

```text
.env
本物のパスワード
有効なセッションID
APIキー
アクセストークン
秘密鍵
個人情報入りのDB
実在人物の画像
```

誤って秘密情報をコミットした場合は、ファイル削除だけで終わらせず、認証情報の失効・再発行とGit履歴への対応を行います。

## セキュリティテストの考え方

自動スキャンだけに依存しません。

アプリ固有の認可、所有権、状態遷移、再送、途中失敗は、要件を理解した人が検査する必要があります。

```text
自動テスト
＋ 静的解析
＋ 脆弱性検査
＋ Burp Suiteによる手動検査
＋ 設計レビュー
```

## 法的・倫理的な範囲

Burp Suiteや攻撃手法は、自分が所有するシステム、または明示的に許可された環境だけへ使用します。

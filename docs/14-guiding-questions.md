# このプロジェクトで大切にする問い

実装中は、常に次の問いを使います。

```text
今回の要求と対象外は明確か
この設計で解決したい問題は何か
AIへ聞く前に自分の案を作ったか
自分の疑問を自分の言葉で説明できるか
他にどのような設計案があるか
この外部ライブラリは本当に必要か
今回の学習対象をライブラリが隠していないか
現在のPhaseに対して過剰な設計になっていないか
責務は適切な場所に置かれているか
依存関係の向きは説明できるか
データの所有者と整合性ルールは明確か
失敗時にどこまで処理を戻す必要があるか
このHTTPメソッドは適切か
このステータスコードは適切か
このリクエストをcurlで説明できるか
Brunoが自動で追加した情報を把握しているか
Brunoだけで確認を終わらせていないか
入力値をどこまで信用してよいか
クライアントが値を書き換えたらどうなるか
未ログイン状態ではどうなるか
別のユーザーが同じIDを指定したらどうなるか
同じリクエストを何度も送ったらどうなるか
処理の途中で失敗したらどうなるか
ログに秘密情報が出ていないか
Burp Suiteで改変された場合も安全か
どのテストを書けば再発を防げるか
実装前の予想と実際の結果はどこが違ったか
今回、何を新しく理解できたか
まだ説明できないことは何か
どの判断で迷い、何を比較したか
AIの提案をそのまま採用していないか
AIの提案を採用・不採用にした理由を説明できるか
AIなしで同じ設計と実装を説明できるか
次のPhaseへ何を持ち越すか
予定と実績に差が生じた理由は何か
次のPhaseの見積もりへ何を反映するか
```
